3. 实施计划¶

3.1 时间安排¶

  • 准备阶段：环境与权限准备（SharePoint 目标库、版本策略、权限模型）；ETL 配置固化（主配置与环境覆盖）；异常规则库与阈值初版。
  • 实施阶段：本地 ETL 执行、Parquet 产出与压缩存档、上传 SharePoint 并启用版本控制；Power BI 指向 Parquet 数据源；报表异常高亮规范统一。
  • 验证阶段：发布前一致性回放门禁；抽样对照 DAX/ETL 误差 ≤ 0.5%；运行稳定性与性能评估（失败率、刷新时长）。

3.2 资源分配¶

  • 人员安排：数据工程（ETL）、报表开发（Power BI）、平台运维（SharePoint/权限）、业务口径与测试。
  • 设备需求：ETL 运行环境（本地/Server）、SharePoint 存储空间、报表工作区容量。
  • 预算分配：存储与租户资源、自动化与告警服务（如邮件/Teams）。

6. 验证方法¶

  • 验证指标：刷新失败率、刷新时长、DAX/ETL 误差、异常闭环时效、数据量与文件体积变化。
  • 数据收集：ETL 日志、Power BI 刷新日志、SharePoint 版本记录、质量报告。
  • 效果评估：与改进前对比（处理位置、格式、性能与体积）；参考 etl/index.md 的 Parquet 性能提升与压缩率区间（读取速度 +90%-95%，体积 -84%-90%）。
