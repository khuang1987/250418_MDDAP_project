定义阶段（Define）¶

本页整合“定义阶段”相关文件为单页报告，明确项目愿景、问题陈述、目标与范围，并指出后续阶段的衡量与约束。SA/SFC 报表仅作为技术试点，最终目标是支撑多部门、多主题的运营数据平台与指标体系。

1. 项目背景¶

MDDAP 发现的改善机会：组织内使用多个管理与运营工具（Microsoft Project、Planisware、Teams Planner、Excel、Power BI、OEE、SFC、MES 等），数据口径不一致、分散在多处，整合与对账依赖人工，效率低且容易出错。

2. 问题陈述¶

1) 运营数据分散、缺乏标准化，人工采集与整合成本高、易出错。
2) 各系统间存在数据差异，缺乏统一口径与对账，难以及时发现与纠正。
3) 管理层与工程师难以及时获取所需数据，影响决策与执行效率。

对应业务痛点：项目跟踪、资源分配、运营分析效率低；数据采集、对账缺少自动化；持续改进缺乏可靠的数据基础。

3. 项目目标（Platform Goals）¶

  • 建立统一的数据处理与指标口径平台，覆盖供应链、生产、质量、CI 等部门。
  • 形成“配置中心 + 日历服务 + 标准时间接口 + 字典表”的统一治理能力。
  • 建立数据一致性校验体系与发布门禁，确保指标口径统一与可追溯。
  • 开发统一的可视化发布能力，支持实时或准实时的分析与决策。

4. 项目输出（Y 值）¶

  • 核心 Y：运营数据准确率（目标：100%）
  • 定义：Dashboard 展示数据与实际业务数据一致性。
  • 计算：准确数据项数 / 总数据项数 × 100%。
  • 要求：面向平台的全域数据在既定时效内可用，且准确率达标。

5. 范围与不在范围¶

范围：
- 覆盖运营相关数据域（项目进展、资源分配、运营指标、收益等）。
- 涵盖 Planisware、Project、Excel、OEE、SFC、MES、Planner 等系统与线下表格数据。
- 涉及数据标准、采集、校验、展示及监控等流程。

不在范围：
- 现有业务系统的底层架构改造。
- 非运营相关的日常行政数据。

6. 成功标准与里程碑¶

  • 指标口径统一并形成文档化标准；平台具备配置化与版本化治理能力。
  • 通过试点（SA/SFC）验证结构可复用、口径一致性可回放，刷新稳定性达到目标。

7. 关系与后续¶

  • 测量阶段将基于现状度量形成问题清单与影响评估（见 project/02_Measure_测量阶段/index.md）。
  • 分析与改进阶段输出平台级鱼骨、流程与对策（见 project/analyze.md、project/improve.md）。
  • 控制阶段固化门禁与监控机制，保障长期有效（见 project/05_Control_控制阶段/index.md）。
