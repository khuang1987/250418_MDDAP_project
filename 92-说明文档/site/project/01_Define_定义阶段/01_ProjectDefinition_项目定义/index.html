项目定义 / Project Definition¶

中文部分¶

标题¶

运营数据整合与智能报表项目

背景¶

MDDAP发现的改善机会：改善项目使用多个管理工具（Microsoft Project、Planisware、Teams Planner），导致数据不一致和管理效率低下。

问题陈述¶

  1. 运营数据分散，缺乏标准化，数据采集依赖人工，效率低且易出错。
  2. 各系统间数据存在差异，难以及时发现和纠正，影响数据的准确性和一致性。
  3. 管理层和工程师难以及时获取所需的项目及运营数据，影响决策和执行。

业务痛点¶

  • 数据分散且不一致，导致项目跟踪、资源分配和运营分析效率低下，影响决策质量。
  • 数据获取依赖人工，缺乏自动化工具支持，增加管理成本，阻碍持续改进。

项目目标¶

  • 建立统一的运营数据标准，包括：
  • 明确定义每个数据项的数据来源系统
  • 制定标准化的数据采集算法和计算规则
  • 设定数据更新频次和时效性要求
  • 建立数据质量评估标准
  • 构建数据一致性校验体系，确保数据准确可靠。
  • 开发统一的运营数据可视化平台，支持实时分析和决策。

项目输出（Y值）¶

Y：运营数据准确率（目标：100%） - 定义：Dashboard中展示的运营数据与实际业务数据的一致性程度。 - 计算方式：准确数据项数 / 总数据项数 × 100% - 要求：所有运营数据在Dashboard中实时展示，且准确率达到100%。

范围¶

  • 所有运营相关数据，包括项目进展、资源分配、运营指标和收益等。
  • Planisware、Project、Excel、OEE、SFC、Planner等系统数据及线下表格数据。
  • 数据标准、采集、校验和展示相关流程。

不涉及范围¶

  • 现有业务系统的底层架构改造。
  • 非运营相关的日常行政数据。

English Section¶

Title¶

Operational Data Integration and Visualization Improvement

Background¶

Improvement opportunities identified by MDDAP: The use of multiple management tools (Microsoft Project, Planisware, Teams Planner) has led to data inconsistency and low management efficiency.

Problem Statement¶

  1. Operational data is fragmented and lacks standardization; data collection relies on manual work, which is inefficient and error-prone.
  2. Data discrepancies exist between different systems, making it difficult to identify and correct issues in time, affecting data accuracy and consistency.
  3. Management and engineers cannot efficiently access the required project and operational data, impacting decision-making and execution.

Business Pain Points¶

  • Scattered and inconsistent data leads to inefficient project tracking, resource allocation, and operational analysis, affecting decision quality.
  • Manual data acquisition without automated tools increases management costs and hinders continuous improvement.

Project Goals¶

  • Set up unified operational data standards, including:
  • Clear definition of data source systems for each data item
  • Standard data collection methods and calculation rules
  • Data update frequency and time requirements
  • Data quality check standards
  • Build a data check system to ensure data is correct and reliable
  • Create a unified data display platform for real-time analysis and decision-making

Project Output (Y Value)¶

Y: Operational Data Accuracy Rate (Target: 100%) - Definition: The consistency level between operational data displayed in Dashboard and actual business data. - Calculation: Number of accurate data items / Total number of data items × 100% - Requirement: All operational data must be displayed in real-time on Dashboard with 100% accuracy.

Scope¶

  • All operational data, including project progress, resource allocation, operational metrics, and benefits.
  • Data from systems including Planisware, Project, Excel, OEE, SFC, Planner, and offline spreadsheets.
  • Data standards, collection, validation, and presentation processes.

Exclusions¶

  • Underlying architecture changes of existing business systems.
  • Non-operational daily administrative data.
